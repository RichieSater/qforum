Jamie McCorriston,

What I see from attached backtest :
Returns are adjusted but not prices.
Missing 1:5 split price adjustment.

What about this:
https://www.quantopian.com/help#overview-datasources:

When your algorithm calls for historical price or volume data, it is adjusted for splits, mergers, and dividends as of the current simulation date. In other words, if your algorithm asks for a historical window of prices, and there is a split in the middle of that window, the first part of that window will be adjusted for the split. This adjstment is done so that your algorithm can do meaningful calculations using the values in the window.

Simon Thornington

Bug notwithstanding, I think part of Q2 was changing history so it only adjusted up to the point in time of the simulation. So, if there is a split on the 10th, nowadays you should expect the output of history as simulated on the 9th to differ from the output of history as simulated in the 10th.

Simon Thornington

An under appreciated (in my opinion) consequence of this is that any code which stores prices in context, or accumulates price-derived data in context, is wrong. It always was, since live trading splits would break it the same way, but now it's clear in backtests too.
