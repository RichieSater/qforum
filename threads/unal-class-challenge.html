We can't find the opt.MaxTurnover(0.2) constrain in the Optimize Reference but my students are using it of because it is explained in https://www.quantopian.com/tutorials/getting-started#lesson7 and it seems to work. But in https://www.quantopian.com/posts/optimize-api-maxturnover-constraint-is-it-supported Scott Sanderson mentions that "We don't have have any immediate plans to remove or deprecate MaxTurnover, but we haven't been focusing on it much because it's not clear whether using a hard constraint (and all Optimize API constraints are hard constraints right now) is actually a reasonable strategy for limiting turnover. In particular, when combined with other constraints, it's pretty easy for MaxTurnover to cause mutually-unsatisfiable sets of portfolio constraints (for example, if you have a tight DollarNeutralconstraint and a MaxTurnover constraint, you can end up infeasible if your portfolio shifts too far away from dollar neutral between orders)."

Should I recommend them to handle that constrain with their own code or is it ok to use here.
